### イベントハンドラに関数を渡すときの落とし穴
正しい形
```javascript
<button onClick={handleClick}>
```
間違った形
```javascript
<button onClick={handleClick()}>
```
上の方はhandleclick関数がonClickイベントハンドラとして渡されている。これにより、ユーザーがボタンをクリックした時にのみコールするようになっている
下のほうだと、末尾に()があるため、レンダーの際にすぐに関数が実行されてしまう

### イベント伝播
- イベントは子要素から順に実行されていく
- Reactでは<font color="#ff0000">onScroll以外</font>のすべてのイベントが伝播する
**デフォルト動作を防ぐ**:ブラウザのイベントには、デフォルトの動作が関連づけられているものもある。例として、formのsubmitイベントは、その中のボタンがクリックされるとデフォルトではページ全体がリロードされる

## e.stopPagenation()とe.preventDefault()は異なる

- e.stopPropagation() は、ツリーの上側にあるタグにアタッチされたイベントハンドラが発火しないようにします。
- `e.preventDefault()は、数は少ないですがイベントがブラウザデフォルトの動作を持っていた場合に、それを抑制します。


# Headのタグは重複してはいけない
key:同じものがあった場合に上書きするための名前たぐ
```typescript
<Head>
  <meta property="og:title" content="最初のタイトル" key="title" />
</Head>
<Head>
  <meta property="og:title" content="新しいタイトル" key="title" />
</Head>
```
同じキーを持つタグは後の方だけが採用される。
Next.jsは<span style="background:#fff88f">複数のコンポーネント</span>があるから、タグが重複することがよくある
###  結論：**OGP や title には必ず key を付けるべき**
