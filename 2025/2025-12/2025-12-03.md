event.preventDefaultメソッドは、submitイベントの発生元であるフォームが持つデフォルトの動作をキャンセルするメソッドです。
以下が使用例
```typescript
        onKeyDown={(e) => {
          if (e.key === 'Enter') {
            e.preventDefault()
          }
        }}
```

`onKeyDown`は、Reactのイベントハンドラで、キーボードのキーが押されたときに発火するイベントです。このイベントハンドラは、主に入力フォームやホットキーなど、キーボードの操作に応じて特定のアクションを実行したい場合に使用されます。

# ユーザー権限について(高岡さんのNotionより

## ADR(Architecture Decision Record)
:ソフトウェアやシステムのアーキテクチャにおける重要な意思決定、理由、結果などを文書化するための手法

RBAC（ロールベースアクセス制御）:権限は「ユーザー」ではなく「ロール」に紐付け。ユーザーは自分に割り当てられたロールを通じて必要な権限を得る

社内管理画面のアクセス制御をするための技術としてPunditを使用

## Authorization
アクセス制御：ユーザーが指定されたリソースとアクションにアクセスできるかどうかを返す
```typescript
authProvider.canAccess({ resource, action })
```
権限：コンポーネントが検査できる権限リストを返す
```typescript
authProvider.getPermissions()
```

###  Storage: setItem() メソッド

```js
setItem(keyName, keyValue)
```
keyName:作成または更新したいキーの名前を持つ文字列
keyValue:作成または更新したいキーに対して渡したい値を持つ文字列

<span style="background:#fff88f">メソッドにおけるシグネチャとは</span>：メソッド（関数）が “どんな形で呼ばれ、何を返すべきか” を表した約束ごと（設計図のようなもの）。1つのメソッドは基本的に1つのシグネチャを持っている
