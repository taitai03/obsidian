# Rakeタスクではreturnを使えない

前提
Rakeタスクは「メソッド」ではなく「ブロック」である

Rubyにおいて、
- returnは「メソッド」を抜ける
- nextは「ブロック」を抜ける

### ブロックとは
ブロックとは、処理の集合体。ブロックはメソッドに渡すと言う形でしか存在できない

### メソッドがブロックを受け取る方法

①yield
```ruby
def hoge
  yield(100)
end

hoge { |e| e + 1 } # => 101
```

ちなみに|e|は「受け取る値の名前」。「外から渡されてくる値を e という名前で受け取ります」と言う意味が

```ruby
 |x| puts x 
```

{ |x| puts x }

# paramsについて

### 概要
paramsはHTTPリクエストのパラメーーターを表すハッシュライクなオブジェクトで、コントローラー内でユーザーからの入力データを扱う際に中心的な役割を果たす

### HTTPリクエストとパラメーターの関係性
Webアプリケーションでは、ユーザーがブラウザを通じてサーバーにリクエストを送信する。そのリクエストには以下のような様々な形でデータが含まれる可能性がある
- GETリクエストのクエリ文字列
- POSTリクエストのフォームデータ
- URLに含まれるパラメータ

Railsは、これらのデータを自動で解析し、paramsオブジェクトとしてまとめ上げる。
これにより、データのでどころに関わらず、統一された方法でアクセスできるようになる


### paramsの基本的な使い方

```ruby
class UsersController < ApplicationController
  def show
    @user = User.find(params[:id])
    # params[:id]はURLから取得されたユーザーID
  end

  def create
    @user = User.new(params[:user])
    # params[:user]はフォームから送信されたユーザー情報
  end
end
```
pramsは文字列のキーを持つハッシュのように振る舞うが、実際には、ACtionController::Parametersのインスタンスである。これにより、文字列キーとシンボルキーの両方でアクセスできる便利さと、セキュリティの機能が実装されている。
```ruby
# 以下は同じ値にアクセスします
params['user_id']
params[:user_id]
```

ただ、一つ注意する点として、paramsの値は摘手に文字列であると言うことです。<span style="background:#fff88f">数値や真偽値を使う際は型変換が必要になる</span>

# 正規表現
　「パターンを指定して、文字列を効率よく検索、置換するためのミニ言語」

\d　は正規表現で特別な意味を持つ。このような特別な意味を持つ文字を「メタ文字」という

\d は一個の半角数字を意味するメタ文字


名前：伊藤淳一
電話：03-1234-5678
住所：兵庫県西脇市板波町1-2-3

これに対し、`\d\d-\d\d\d\d-\d\d\d\d`を用いると、電話の部分がマッチする
-はメタ文字でない
