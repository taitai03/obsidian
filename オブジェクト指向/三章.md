# 3.1依存関係を理解する

###### 依存関係を意識する

以下の時、オブジェクトには依存関係がある
- 他のクラスの名前がある
- self以外のどこかに送ろうとするメッセージがある
- メッセージが要求する引数がある
- それらの引数に順番がある

不必要な依存はコードの<font color="#ff0000">「合理性」を損なう</font>
→それぞれのクラスが持つ<span style="background:#fff88f">依存関係を必要最低限</span>のものにする必要がある

##### オブジェクト間の結合(CBO:Coupling Between Objects)
- オブジェクトの結合が依存関係を作り出す

依存関係によって、複数のオブジェクトはあたかも一つのオベジェクトのように振る舞うため、<span style="background:#fff88f">変更が同時に襲ってくる</span>ことがある

##### ほかの依存関係
<span style="background:#b1ffff">破壊的な類の依存が生じる場合</span>
- 何かを知るオブジェクトを知るオブジェクトを知るオブジェクトがある時
- メッセージチェーンは依存関係を作る
<span style="background:#b1ffff"> コードに対するテストの依存関係</span>
- テストはコードを参照するが故にコードの依存する
- テストからコードへの過度な結合は、コードからコードへの過度な結合と同じ結果をもたらす

<u>インスタンス変数の作成を分離する</u>

```ruby
class Gear
	attr_reader :chainring, :cog, :rim, :tire
	def initialize(chainring, cog, rim, tire)
		@chainring = chainring
		@cog = cog
		@wheel = Wheel.new(rim, tire)
	end

	def gear_inches
		ratio * wheel.diameter
	end
```

```ruby
class Gear
	attr_reader :chainring, :cog, :rim, :tire
	def initialize(chainring, cog, rim, tire)
		@chainring = chainring
		@cog = cog
		@rim = rim
		@tire =tire
	end

	def gear_inches
		ratio * wheel.diameter
	end
	
	def wheel
		@wheel || =Wheel.new(rim,tire)
	end
```

どちらもGearは知りすぎているが、gear_inches内の<span style="background:#fff88f">依存数は減り</span>、同時に、GearがWheelに<span style="background:#fff88f">依存していることが明確</span>になった

# 3.3依存方向の管理
依存関係には方向がある

##### 依存方向の逆転

依存関係の方向に関する決断は、将来にわたる影響を及ぼし、その影響は、アプリケーションの寿命として現れる。間違った決断を下してしまうと、依存関係に徐々に侵食され、変更は難しくなる
